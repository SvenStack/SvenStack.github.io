üí° Intuition & Brute Force

A naive recursive solution tries all possibilities for ‚Äò*‚Äô (match zero, one, multiple). This leads to exponential time in the worst case.

Key observation for DP: define dp[i][j] = whether s[0..i) matches p[0..j). Build up using smaller prefixes.

‚úÖ Optimized Solution (Dynamic Programming)

Transitions:

  1. If p[j-1] is a normal char or ‚Äò.‚Äô, then:
    dp[i][j] = dp[i-1][j-1] && (p[j-1] == s[i-1] || p[j-1] == ‚Äò.‚Äô)

  2. If p[j-1] == ‚Äò*‚Äô, it stands for zero or more of p[j-2]:

    • Zero occurrence: dp[i][j] = dp[i][j-2]
    • One or more: if p[j-2] matches s[i-1], dp[i][j] = dp[i-1][j]

Initialize dp[0][0] = true, and handle patterns like a*, ab that can match empty string.
