Algorithm-Day10-Subarray-Sum-Equals-K-lc-560

üß© Problem Description

Given an array of integers nums and an integer k, return the total number of continuous subarrays whose sum equals to k.

Example:

Input: nums = [1,1,1], k = 2
Output: 2

üí° Brute Force (Too Slow)

  • ‚ùå Time complexity: O(n¬≤)
  • Works only for small inputs

üí° Optimized Approach: Prefix Sum + HashMap

‚ú® Key Insight

If we know the prefix sum up to index i is sum, and we previously saw a prefix sum of sum - k,
then the subarray between those two indices sums to k.

‚úÖ Steps

  • Use a Map<Integer, Integer> to count how many times each prefix sum has occurred.
  • At each step:
    • Add current number to running sum
    • Check if sum - k has appeared before ‚Üí it means a subarray with sum = k ends here
    • Add sum to the map

üî¢ Java Code

‚è± Time and Space Complexity

  • Time Complexity: O(n)
    One pass through the array
  • Space Complexity: O(n)
    For the prefix sum map

‚úçÔ∏è Summary

  • Prefix sum is a powerful technique for cumulative problems
  • The hash map tracks all possible sums we‚Äôve seen so far
  • Key trick: initialize the map with {0:1} to handle cases where subarray starts at index 0

This is a high-frequency interview question and a foundation for other problems like Longest Subarray Sum, Count of Subarrays With Sum Divisible by K, etc.

‚úçÔ∏è Summary

üí° Optimized Approach 1: Precomputed Arrays (Prefix Max)

We use two arrays:

  • leftMax[i]: max height to the left (including i)

üî¢ Java Code (Prefix Max)

‚è± Complexity

  • Time: O(n)
  • Space: O(n)

üí° Optimized Approach 2: Two Pointers (Space O(1))

We use two pointers (left, right) and two variables (leftMax, rightMax) to simulate the process:

  • Always move the pointer with lower height
  • Update leftMax or rightMax and calculate trapped water accordingly

üî¢ Java Code (Two Pointers)

‚è± Complexity

  • Time: O(n)
  • Space: O(1)

‚úçÔ∏è Summary

üí° Naive Idea (Two-Pass Write)

  • Create a new array and copy all non-zero values into it, then append zeroes.
  • ‚úÖ Correct but ‚ùå not in-place. So it‚Äôs invalid.

üí° Optimal In-Place Approach (Two Pointers)

üî¢ Java Code

‚è± Time and Space Complexity

  • Time Complexity: O(n) ‚Äì single pass for placement + fill
  • Space Complexity: O(1) ‚Äì in-place manipulation

‚úçÔ∏è Summary

  • The key is to use two pointers to separate writing and reading.
  • Remember: ‚ÄúIn-place‚Äù means no extra array.
  • Common interview problem ‚Äî focus on efficiency and clarity.

üß© Problem Description

Given an unsorted array of integers nums, return the length of the longest consecutive elements sequence.

You must write an algorithm that runs in O(n) time.

Example:

Explanation: The longest consecutive sequence is [1, 2, 3, 4]. Therefore its length is 4.

üí° Brute Force? Not Enough

A naive solution would be to:

  • Sort the array ‚Üí O(n log n)
  • Iterate to find longest streak

This fails the O(n) constraint, so we need a better approach.

üí° Optimized Approach (Using HashSet)

To achieve linear time:

  1. Store all numbers in a HashSet for constant-time lookups.
  2. Iterate through the array, and only start a new sequence if num - 1 is not in the set (i.e., it‚Äôs the start of a sequence).
  3. Expand forward: check how long the consecutive streak goes.
  4. Track the max length.

üî¢ Java Code

‚è± Time and Space Complexity

  • Time Complexity: O(n)
    Each number is added and checked in constant time.
  • Space Complexity: O(n)
    To store all unique numbers in the HashSet.

‚úçÔ∏è Summary

  • This is a great example of using HashSet to eliminate redundant computation.
  • The key optimization is only starting a streak when num - 1 is not in the set.
  • Sorting is too slow for this problem ‚Äî always check time constraints carefully.

This technique of using hash-based lookup to replace sorting is common in many ‚ÄúO(n)‚Äù constraint problems.
